# Spec — Registros Alto Custo (Alto Custo = CEAF)

## Overview

Desktop application for registering and managing patient medication records across four workflow types, with batch-based organization and weekly spreadsheet export. Built in Python with **PySide6** + SQLite, with a layered architecture (database → services → state → GUI).

---

## Data Model

```sql
malotes
  id          INTEGER PRIMARY KEY AUTOINCREMENT
  date        TEXT NOT NULL

pacientes
  id          INTEGER PRIMARY KEY AUTOINCREMENT
  name        TEXT NOT NULL UNIQUE

items_catalog
  id          INTEGER PRIMARY KEY AUTOINCREMENT
  name        TEXT NOT NULL UNIQUE
  unidade     TEXT NOT NULL DEFAULT 'un'

registros
  id            INTEGER PRIMARY KEY AUTOINCREMENT
  tipo          TEXT NOT NULL     -- 'entrada' | 'renovacao' | 'retirada' | 'urgente'
  paciente_id   INTEGER NOT NULL REFERENCES pacientes(id)
  malote_id     INTEGER NOT NULL REFERENCES malotes(id)
  created_at    TEXT NOT NULL
  waiting_docs  INTEGER NOT NULL DEFAULT 0
  UNIQUE (tipo, paciente_id, malote_id)

registro_items
  id            INTEGER PRIMARY KEY AUTOINCREMENT
  registro_id   INTEGER NOT NULL REFERENCES registros(id) ON DELETE CASCADE
  item_id       INTEGER NOT NULL REFERENCES items_catalog(id)
```

Indexes on `registros(malote_id)`, `registros(paciente_id)`, `registros(tipo)`, `pacientes(name COLLATE NOCASE)`, `registro_items(registro_id)`, `registro_items(item_id)`.

## UI Architecture

### Start Page

The permanent home base. Contains:

- **Theme toggle button** — switches between dark/light theme.
- **Active malote display** — shown as a clickable label (e.g. *"08/04/2026"*). Clicking opens the malote dialog.
- **Search bar** — searches across registros in the active malote by patient name. Results show patient name + tipo badge. Clicking a result opens the Entry Page in edit mode.
- **Four tipo buttons** — in a 2×2 grid: Entrada, Renovação, Retirada, Resolver na hora. Each navigates to the Entry Page for that tipo. Requires an active malote.
- **Visualizar Malote button** — navigates to the Preview Page.
- **Exportar Planilha button** — generates and downloads a `.xlsx`. Disabled if no active malote.
- **Gerenciar Listas button** — navigates to the List Manage Page.

### Entry Page

Reached by clicking a tipo button (new record) or a search result (edit mode). Header shows a tipo combo (switchable) and the malote label (switchable).

- **Patient field** — searchable combo with live dropdown filtering `pacientes` by name. Patients are auto-created on save if not found (no separate "Novo paciente" button needed).
- **Items list** — vertical list of searchable combos, each picking from `items_catalog`. Each row has a remove button. *"Adicionar Item"* button adds a new row. When a patient is selected, previous items from that patient's other registros are pre-populated. If an existing registro is found for the same tipo/paciente/malote, it loads into edit mode.
- **Esperando documentos checkbox** — marks the registro as waiting for documents; excluded from spreadsheet export.
- **Ficar nesta tela checkbox** — when checked, the form resets after save instead of navigating back. Default: unchecked. Persisted via config (`stay_on_page`).
- **Status label** — shows "Novo registro" or "Editando registro".
- **Delete button** — visible only in edit mode. Confirms before deleting.
- **Save button** — saves registro and registro_items. Navigates back or resets form depending on stay-on-page toggle.
- **Back button** — returns to start page without saving.

### Malote Dialog

Modal tree view triggered from the malote label. Contains:

- **Tree view** — groups malotes by year (previous years collapsed). Active malote shown with checkmark and bold text. Clicking a malote sets it as active.
- **Right-click context menu** — Edit (opens edit dialog), Delete (only if no registros exist in the malote).
- **Novo Malote button** — opens creation popup.
- **Creation popup** — date input accepting loose formats (`08/04`, `08/04/26`), auto-completes to full date. On confirm: saves malote, sets it as active.

### Preview Page

Tabbed table view of all registros in the active malote:

- **Tabs** — one per tipo, with count in tab label (e.g. "Entradas (5)").
- **Search** — per-tab search bar filtering by patient name or medication.
- **Table** — columns: Nome | Medicamentos (newline-separated). Rows sorted alphabetically by patient name. Double-click a row to open it in the Entry Page.
- **Right-click context menu** — Edit → Tipo (change tipo), Edit → Malote (move to another malote), Edit → Nome do paciente (rename patient).
- **Registros with `waiting_docs=1` are excluded** from this view.

### List Manage Page

Tabbed view for managing medications and patients:

- **Medicamentos tab** — searchable list of all items from `items_catalog`. Add, edit (double-click), delete items. Delete blocked if item is in use.
- **Pacientes tab** — searchable list of all patients. Add, edit (double-click), delete patients. Delete blocked if patient has registros.

|------------|-------------------------------|

| Escape     | Back to start page            |
| Ctrl+D     | Open malote dialog            |
| Ctrl+R     | Focus search field            |
| Ctrl+G     | Open preview page             |
| Ctrl+T     | Open list manage page         |

| Ctrl+W     | Toggle "waiting docs"         |
| Ctrl+Q     | Toggle "stay on page"         |

## Export

Single `.xlsx` file, generated on demand. Structure:

- One tab per tipo: *Entrada*, *Renovação*, *Retirada*, *Resolver na hora*
- Filtered to active malote, excluding `waiting_docs=1` registros
- Header row 1: "USAFA OCIAN" (merged, font 20pt Arial, centered)
- Header row 2: "[TIPO TITLE] - [dd/mm]" (merged, font 16pt Arial, centered)
- Data rows: **Nome** | **Medicamentos** (newline-separated item names), centered with wrap text, thin borders
- Auto-sized columns, A4 portrait, fit-to-width, no gridlines, centered printing
- Filename: `Malote_[dd-mm]_[HHMMSS].xlsx`

## Configuration

Saved to JSON (`data/config.json`):

- `stay_on_page`: bool (default false)
- `theme`: `"dark"` | `"light"`
- `last_malote_id`: int — restores active malote on relaunch
- `save_path`: str — default export destination (user downloads by default)
